Normalising of the Species DB used for the Grok SQL module.

Normalisation is done using Microsoft Access, making use of  its schema page and the query builder.

There is no direct code in the repository, but there are links below to DB versions and Word "How to" files.

1st Normal form involves atomising the scientific name into Genus and Species

2nd Normal form involves the creation of tables

The repository for this site is at: https://github.com/bwattle/arrRecords

This site gives practical examples of the NSW Information Processing and Design syllabus, section 9.2 (Databases) 
Some tasks correspond to the theory in both the Powers and Davis Texts for this subject

This project was inspired by students working on the Grok SQL course, which gives practical examples of SQL for filtering and ordering.

From the initial flat file (single table): 
  • we re-create some of the queries from the Grok SQL course, using the Access query builder and SQL view.
  • create a report with multiple grouping levels.

Access (.accdb) and Word "How to" files are stored in the "Releases" section of GitHub. Suggestions regarding errors would be gratefully received at: neville.goodyer@det.nsw.edu.au.
